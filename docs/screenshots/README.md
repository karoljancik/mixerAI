# Screenshot Checklist

Use these captures when preparing a portfolio page, README update, or job application attachment.

## Primary Screens

1. Signed-out landing page
   Show the React landing surface with the architecture pills and sign-in card.
2. Authenticated workspace hero
   Capture the top metrics and headline after logging in.
3. Transition Lab
   Load two ready tracks so Deck A and Deck B are both populated.
4. Recommendation Engine
   Submit two corpus sets and capture a few ranked recommendations.
5. Mix Analysis
   Upload two local files and capture the BPM/overlay metrics plus waveform markers.
6. Library Operations
   Include a row with a failed analysis so the retry/error state is visible.

## Recording Tips

- Use a seeded/demo account with a small curated track library.
- Prefer 16:9 captures at desktop width.
- Keep filenames stable, for example:
  `landing-page.png`
  `workspace-dashboard.png`
  `transition-lab.png`
  `recommendations.png`
  `mix-analysis.png`
  `library-retry-state.png`

## Demo Video Flow

1. Show the landing page.
2. Sign in.
3. Upload a track or highlight existing ready tracks.
4. Load Deck A and Deck B.
5. Render an AI transition reference.
6. Open recommendations.
7. Run local mix analysis.
8. Close on the architecture story: React SPA, ASP.NET Core BFF, backend API, Python pipeline.
